# 결 (Gyeol) — 핵심 질문 체계 설계 문서 (v7)

## 11. 회피 옵션 (v7 갱신)

본 앱은 *어려운 질문을 마주하지만 도망갈 곳도 두는* 앱이다. 사용자가 답을 할 수 없는 영역을 강제하지 않는다.

### 11.1 "더 쉽게 설명해주세요" (v6 그대로)

질문이 추상적·난해하면 *깊이 단계 낮춤*. 같은 영역의 핵심을 묻되 더 일상적 언어로.

최대 3단계까지. 그 이상에서는 §11.3의 회피 옵션으로 안내.

### 11.2 깊이 메타데이터 (v6 그대로)

쉬워진 질문에 답한 사람과 원 질문에 답한 사람의 답변은 같은 깊이가 아니다. `depth_level: 1/2/3` 메타가 부착되며 매칭 시 깊이 차이를 보정한다.

### 11.3 세 가지 회피 옵션 (v7 갱신 — 사전 고지 포함)

**(1) "더 쉽게 설명해주세요"** — 깊이 단계 낮춤

**(2) "이 영역은 건너뛸게요"** — 영역 전체 스킵.

사유 선택 (사전 정의 enum):
- 공개하고 싶지 않음
- 아직 정리되지 않음
- 중요하지 않다고 판단
- 기타

**v7 사전 고지 추가**: 사유 선택 모달에 다음 안내 명시:

> 선택한 사유는 매칭된 상대에게 노출됩니다.
> 예: *"이 영역은 답변하지 않았습니다 — 사유: 공개하고 싶지 않음"*

이 사전 고지로 사용자가 *건너뛰기 사유 자체가 시그널*임을 인지하고 선택.

자유 텍스트 입력은 받지 않음. *기타* 선택 시도 추가 정보 노출 없음.

**(3) "답변은 하되 이 영역 분석은 비공개로"** — 답변 진행, 본인 자기이해용 보관, 매칭 풀 미노출.

**v7 사전 고지 추가**: 비공개 보관 선택 시 다음 안내 명시:

> 이 영역의 분석 내용은 매칭 상대에게 공개되지 않습니다.
> 다만 추천 후보에게는 *"이 영역은 비공개로 보관됨"* 이라는 상태만 표시됩니다.
> 매칭 풀 자격 산정에서도 발행으로 카운트되지 않습니다.

**근거**: 사용자가 *분석 내용*과 *비공개 사실*을 구분 인지하지 못하면 신뢰가 깨질 수 있음. 비공개 선택 시 *상대에게 비공개 사실은 표시됨*을 명시 안내.

### 11.4 회피 옵션 사전 고지 메커니즘 (v7 신설)

세 옵션 모두 *선택 시점*에 결과를 명시. 사용자가 선택 후 *예상치 못한 노출*을 발견하지 않도록.

**구현 방식**:
- 회피 옵션 모달 디폴트: 선택 옵션 + 선택 결과 한 줄 요약
- *자세히 보기* 펼침 시: 전체 영향 (매칭 풀 자격, 상대 노출, 자기 검토 영역) 안내
- 사용자가 *최초 선택* 시에만 자동 펼침. 같은 옵션 재선택 시는 접힘 디폴트

**예시 — 비공개 보관 모달 (v7)**:

```
[모달]

이 영역의 답변을 비공개로 보관합니다.

이 선택의 영향:
· 분석 내용은 매칭 상대에게 공개되지 않습니다.
· 추천 후보에게 "이 영역은 비공개로 보관됨" 상태만 표시됩니다.
· 매칭 풀 자격 산정에서 이 영역은 발행으로 카운트되지 않습니다.
· 비공개 영역이 늘어날수록 매칭 가능성이 줄어듭니다.

[취소]      [비공개로 보관]
```

### 11.5 음성 입력 옵션 (v7 신설)

본 앱 답변 특성상 *긴 답변 + 사고 정리하면서 천천히 말함*이 흔하다. 키보드 외 음성 입력 옵션 제공.

#### 11.5.1 음성 입력 SDK 결정

**Apple Speech Framework (SFSpeechRecognizer) 메인 + iOS 시스템 받아쓰기 자연 지원**.

근거:
- iOS 10+ 표준 SDK, 한국어 (ko-KR) 지원, 무료
- on-device 모드 (iOS 13+, A9 이상): 네트워크 불필요, 데이터 거주 충족
- 외부 STT API (Google Cloud, Whisper) 거부 — 비용 + 민감 정보 외부 전송 부담

#### 11.5.2 인터랙션 패턴

**디폴트**: 키보드 입력
**음성 옵션**: 입력 영역 우측 상단에 마이크 아이콘 (SF Symbol mic)

**음성 입력 흐름**:
1. 마이크 아이콘 탭 → 권한 요청 모달 (최초 1회)
2. 권한 허용 → 녹음 시작
   - 마이크 아이콘 활성 상태 (펄스 애니메이션)
   - *듣고 있습니다* 배너 + 녹음 시간 + 빨간 점 애니메이션
   - 화면 하단 음성 진폭 시각화 (waveform)
   - 실시간 텍스트 변환 (사용자가 말하는 동안 텍스트 노출)
3. 녹음 종료 (사용자 *녹음 종료* 버튼 또는 60초 무음 자동)
4. 변환된 텍스트가 입력 영역에 들어감
5. 사용자가 키보드로 자유롭게 수정 가능

#### 11.5.3 1분 제한 우회

Apple Speech Framework는 세션당 1분 제약.

**우회 메커니즘**:
- 1분 가까워지면 *자동 세션 재시작* + 누적 텍스트 유지
- 사용자에게는 *연속 녹음*으로 보임
- 또는 사용자가 멈춤 → 다시 마이크 탭으로 재시작 (자연스러움)

#### 11.5.4 음성 데이터 처리 정책

- on-device 모드 사용 → 음성 데이터 외부 전송 없음
- 변환된 텍스트만 본 앱 백엔드(Supabase)로 전송
- 사용자 개인정보 처리방침에 *음성 처리는 기기 내에서 이뤄지며 음성 자체는 외부로 전송되지 않음* 명시

#### 11.5.5 권한 처리

- iOS 권한: `NSSpeechRecognitionUsageDescription`, `NSMicrophoneUsageDescription` Info.plist 필수
- 권한 거부 시: 마이크 아이콘 비활성, 키보드 입력만 가능
- 권한 거부 사용자에게 *재요청* 강요 안 함 (선택의 영역)

## 13. 개인정보 보호와 민감정보 처리 (v7 갱신)

본 앱이 수집하는 데이터의 성격상 한국 개인정보보호법(PIPA) 23조 민감정보 처리 원칙이 직접 적용된다.

**(1) 별도 동의** — 일반 회원가입 동의와 분리된 민감정보 처리 동의 (v6 그대로).

**(2) 최소 수집 / 목적 제한** — 매칭 외 어떤 목적으로도 사용 금지 (v6 그대로).

**(3) 암호화** — Raw 답변, 분석문, core_identity, explicit_dealbreakers 모두 저장·전송 시 암호화 (v6 그대로).

**(4) 삭제권** — 사용자는 언제든 영역별·전체 데이터 삭제 가능 (v6 그대로).

**(5) AI 학습 사용 금지 고지** — 본 앱 답변 데이터는 외부 AI 학습 데이터로 사용되지 않음 (v6 그대로).

**(6) 비공개 옵션** — §11.3의 세 번째 회피 옵션은 민감정보 보호 원칙의 자연스러운 연장.

**(7) 음성 데이터 처리 (v7 신설)** — Apple Speech Framework on-device 모드 사용. 음성은 사용자 기기 내에서 직접 텍스트로 변환되며 외부 서버로 전송되지 않음. 변환된 텍스트만 본 앱 백엔드로 전송. 동의서에 명시.

**(8) Raw quote 격리 (v7 신설)** — 사용자 답변의 직접 인용은 *answer_evidence* 영역에 격리. 매칭 상대에게 노출되는 *summary*에는 raw quote가 포함되지 않음 (AI 프롬프트 v7 §1.3, §2.1 참조).

## 14. v6 → v7 변경 사항

### 14.1 회피 옵션 사전 고지 (§11.3, §11.4)

v6: 사유·비공개 선택의 *결과*가 명시되지 않음
v7: 모달에 *상대 노출 영향*을 명시. 사용자가 *예상치 못한 노출*을 발견하지 않도록.

### 14.2 음성 입력 옵션 신설 (§11.5)

Apple Speech Framework 기반. on-device 모드 + 1분 제한 우회 + 키보드 자유 수정.

### 14.3 음성 데이터 처리 명시 (§13)

PIPA 동의서에 *음성 데이터 on-device 처리 + 외부 전송 없음* 명시.

### 14.4 Raw quote 격리 명시 (§13)

AI 프롬프트 v7과 정합. summary는 public_safe, raw quote는 answer_evidence 격리.

### 14.5 변경 없는 영역

- §1 본 앱 정체성
- §2-§9 6영역 핵심 질문 (각 영역의 오픈 질문, lens 도구, 후속 질문 메커니즘)
- §10 명시 Dealbreaker 입력 (v6 신설 영역 그대로)
- §12 온보딩 게이트
- 분석 톤 원칙
- 출력 데이터 노출 범위 일반

본 v7 standalone 통합 문서는 다음 사이클에서 작성. 본 사이클은 *v6 + 변경분*으로 운용.
