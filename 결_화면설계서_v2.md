# 결 (Gyeol) — 화면 설계서 (v2)

> 본 문서는 v1에 (1) 음성 입력 옵션 추가, (2) 비공개·건너뛰기 영역 사전 고지 강화를 반영한 v2다. v1의 13개 화면 본문은 그대로 유지하고, 변경 영역만 본 v2에 명시.

> v1 → v2 변경 영역: §5 화면 4 (답변 입력) — 음성 입력 4단계 분리, §7 화면 6 (후속 질문) — 마이크 버튼 추가, §10 화면 9 (명시 Dealbreaker 입력) — 변경 없음 (이미 v1에서 명시), §4 회피 옵션 모달 — 사전 고지 강화. 나머지 화면은 v1 그대로.

> 본 v2 standalone 통합 문서는 다음 사이클에서 작성. 본 사이클은 *v1 + 변경분*으로 운용.

> **v1 → v2 주요 변경**: §3 참조.

## 1. 변경 영역 1 — 화면 4 답변 입력 (음성 입력 4단계)

v1 §5의 단일 화면 4를 *음성 입력 인터랙션 흐름의 4단계 화면*으로 확장.

### 1.1 화면 4 (디폴트 키보드 입력)

v1 화면 4 본문 + 다음 추가:
- 입력 영역 우측 상단에 *마이크 아이콘* (SF Symbol mic, 회색 배경 위 비활성 아이콘)
- 사용자가 마이크 탭 시 → 화면 4·A 또는 4·B로 전환

### 1.2 화면 4·A — 음성 입력 권한 요청 (v2 신설)

**목적**: 처음 마이크 탭 시 iOS 시스템 권한 모달 노출.

**구성**:
- 모달 타이틀: *"'결'이 음성 인식에 접근하려고 합니다"*
- 모달 본문: *"음성으로 답변을 입력할 수 있도록 마이크와 음성 인식 사용을 허용해주세요. 음성은 기기에서 직접 처리되며 외부로 전송되지 않습니다."*
- 액션: *허용 안 함* / *허용*

**근거**: PIPA 23조 민감정보 처리 동의와 별개로, 마이크 권한·음성 인식 권한은 iOS 시스템 권한. 사용자에게 *기기 내 처리* 명시로 신뢰 확보.

### 1.3 화면 4·B — 음성 입력 녹음 중 (v2 신설)

**목적**: 사용자가 음성으로 답변 중. 시각 피드백 충분히 제공.

**구성**:
- 마이크 아이콘이 *활성 상태*: 강조색 채움 + 펄스 애니메이션 (1.6초 cycle)
- *듣고 있습니다* 배너: 빨간 점 펄스 애니메이션 + 텍스트 + 녹음 시간 표시 (00:23 형태)
- 입력 영역: 실시간 변환 텍스트 + 깜빡이는 커서
- 화면 하단: 음성 진폭 시각화 (waveform, 25개 막대)
- *답변 완료* 버튼이 *녹음 종료* 버튼으로 변경 (사각 stop 아이콘)

**1분 제한 우회**:
- 1분 가까워지면 자동 세션 재시작
- 사용자에게는 *연속 녹음*으로 보임
- waveform과 시간 표시는 누적 시간 표시

**60초 무음 자동 종료**:
- 60초 무음 시 자동 종료
- 종료 시 안내 토스트: *"60초 무음으로 종료되었습니다"*

### 1.4 화면 4·C — 음성 입력 종료, 편집 가능 (v2 신설)

**목적**: 음성 변환 결과를 사용자가 검토·수정.

**구성**:
- 마이크 아이콘 다시 비활성 상태
- 입력 영역에 변환된 텍스트 그대로 들어감
- 키보드 노출 가능 (사용자 탭 시)
- 도움 문구: *"음성으로 받은 텍스트입니다. 필요하면 자유롭게 수정해주세요."*

**자유 결합 가능**:
- 음성 → 키보드 수정 → 다시 음성 추가 모두 가능
- 텍스트 누적 처리

### 1.5 화면 4 디자인 토큰

마이크 버튼:
- 크기: 40×40
- 배경 (비활성): `bg.subtle`
- 배경 (활성): `accent.primary`
- 아이콘 색 (비활성): `text.secondary`
- 아이콘 색 (활성): `bg.primary`
- 펄스 애니메이션: 1.6초 ease-in-out infinite

녹음 배너:
- 빨간 점: `#C44545`, 8×8, pulse 1.2초
- 배경: `bg.subtle`
- radius: 12pt

Waveform:
- 막대 25개, 너비 2.5px, gap 3px
- 색: `accent.primary`, opacity 0.7
- 애니메이션: 1.2초 ease-in-out infinite, 막대별 delay 0.0-0.3초

## 3. 변경 영역 3 — 회피 옵션 모달 (사전 고지 강화)

v1 §4.5의 *건너뛰기 사유 선택 모달*과 §11.6의 *재시작 보조 메뉴*에 사전 고지 강화.

### 3.1 건너뛰기 사유 선택 모달 (v2 강화)

v1 그대로 + 다음 명시 추가:

```
[건너뛰기 모달 — v2]

이 영역을 건너뛰는 사유를 선택해주세요.

◯ 공개하고 싶지 않음
◯ 아직 정리되지 않음
◯ 중요하지 않다고 판단
◯ 기타

──────

선택한 사유는 매칭된 상대에게 노출됩니다.
예: "이 영역은 답변하지 않았습니다 — 사유: 공개하고 싶지 않음"

[취소]      [확인]
```

**v1과 차이**: 사유 노출 영향을 *예시 포함*으로 명시. 사용자가 *어떻게 보이는지*까지 인지하고 선택.

### 3.2 비공개 보관 모달 (v2 신설)

v1에서 *재시작 보조 메뉴*에 *이 영역만 비공개로 보관* 옵션이 있었으나 사전 고지 부족. v2에서 별도 모달:

```
[비공개 보관 모달 — v2 신설]

이 영역의 답변을 비공개로 보관합니다.

이 선택의 영향:
· 분석 내용은 매칭 상대에게 공개되지 않습니다.
· 추천 후보에게 "이 영역은 비공개로 보관됨" 상태만 표시됩니다.
· 매칭 풀 자격 산정에서 이 영역은 발행으로 카운트되지 않습니다.
· 비공개 영역이 늘어날수록 매칭 가능성이 줄어듭니다.

[취소]      [비공개로 보관]
```

**근거**: 핵심질문체계 v7 §11.3의 사전 고지 요구사항을 UI에서 구현.

### 3.3 모달 디자인 토큰

- 배경: `bg.elevated`
- 모달 너비: 320pt (좌우 24pt 마진)
- radius: 18pt
- padding: 24pt
- 타이틀: SemiBold 16pt, `text.primary`
- 본문: Regular 13.5pt, `text.secondary`, line-height 1.55
- 액션 버튼:
  - 좌측 (취소): `text.secondary`, Medium 15pt
  - 우측 (확인/비공개로 보관): `accent.primary`, SemiBold 15pt
  - 구분선: `divider` 1pt

## 4. 변경 영역 4 — 후보 카드 펼침 화면 (변경 없음, 명시만)

v1 §13의 화면 12 후보 카드 펼침에서 *비공개 영역*과 *건너뛰기 영역* 처리는 이미 v1에 정의됨. v2에서 변경 없음. 단 다음 정합성 명시:

**비공개 영역 표시**:
> *"이 영역은 비공개로 보관되었습니다."*
> 
> (사용자 사전 고지: 이 표시는 §3.2 비공개 보관 모달에서 안내된 결과)

**건너뛰기 영역 표시**:
> *"이 영역은 답변하지 않았습니다 — 사유: {skip_reason}"*
>
> (사용자 사전 고지: 이 표시는 §3.1 건너뛰기 모달에서 안내된 결과)

사용자가 자기 검토 화면(화면 10)에서 동일한 표시 미리보기 가능. 매칭 상대에게 노출되는 모습을 발행 전 확인할 수 있도록.
